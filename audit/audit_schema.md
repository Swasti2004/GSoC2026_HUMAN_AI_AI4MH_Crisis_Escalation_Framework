# Audit Schema (Model v1.1)

## Purpose

This document defines the structured audit record format generated by the Crisis Escalation Framework.

All decisions are logged in JSON Lines format:

logs/audit_log.jsonl

Each line represents a single evaluation decision.

---

## Schema Definition

{
  "county_id": "string",
  "timestamp": "ISO8601 string",
  "sentiment_z": "float | null",
  "volume_z": "float | null",
  "geo_score": "integer | null",
  "crisis_score": "float | null",
  "confidence": "HIGH | LOW",
  "escalation_level": "integer (0–3)",
  "model_version": "string"
}

---

## Field Definitions

county_id
Unique identifier for the evaluated county.

timestamp
UTC timestamp of evaluation.

sentiment_z
Z-score deviation from baseline sentiment.

volume_z
Z-score deviation from baseline activity volume.

geo_score
Spatial reinforcement score based on neighboring counties.

crisis_score
Composite weighted crisis score.

confidence
Derived from sample size and statistical stability.

escalation_level
Tiered escalation decision (0–3).

model_version
Version identifier of scoring logic.
